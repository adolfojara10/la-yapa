La Yapa — Session Summary
Built: A working pnpm/Turborepo monorepo with three apps and two shared packages, plus full design system and CI.
Apps: apps/api (Django 5.1 + DRF + Postgres 16, split settings, custom User model with initial migration, health check, JWT auth scaffolding, OpenAPI docs). apps/mobile (Expo SDK 51 + Expo Router + TanStack Query, ThemeProvider with light/dark, 11 components: Button/Input/Card/Badge/Avatar/Modal/BottomSheet/Toast/Skeleton/Icon/Text, in-app /design-system showcase). apps/admin (Next.js 14 + Tailwind + next-themes + 8 shadcn-style Radix primitives).
Packages: @layapa/ui (single-source design tokens: palette, light/dark schemes, type scale, spacing, radii, shadows, motion); @layapa/shared-types (User/Business/Bag/Order/API types).
Key decisions: Tokens-only shared package (no cross-platform components); semantic colors via CSS variables on web + theme context on mobile; placeholder Yapi mascot (8 states) + logo (4 variants) as inline SVGs; Husky handles JS only, pre-commit framework for Python; CI runs JS + Django jobs in parallel with service containers.
